## 卡片一：安全合规 · 12/12

### PPT 数字含义
**12/12** = 工信部《生成式人工智能服务管理暂行办法》及配套文件中列出的 12 项企业 AI 使用合规要求，woclaw 全部覆盖，一项不漏。

### 说辞
> "安全合规这件事，不是有没有做，而是能不能被查到。
> 工信部对企业用 AI 有 12 项明确要求，市面上大多数产品只做了其中几项，woclaw 是 12/12 全覆盖。
> 具体体现在三点：
> 第一，**数据本地**——所有业务数据、对话记录、知识库，100% 存在您自己公司的服务器上，我们的服务器看不到一条数据；
> 第二，**全程可查**——每一次 AI 操作都有日志，本地存储，自己审计，监管来了随时调取；
> 第三，**人工确认**——凡是删除文件、发送消息、调用外部接口这类危险动作，必须有人工二次确认，AI 不能自己跑偏。"

### 深问备答

**Q：12/12 是哪 12 项？你们怎么证明全覆盖？**
> woclaw 对应的合规文件包括：
> 1. 《生成式人工智能服务管理暂行办法》（2023）
> 2. 工信部《互联网信息服务算法推荐管理规定》
> 3. 国家标准 GB/T 35273《个人信息安全规范》
>
> 12 项核心要求涉及：数据不出境、用户数据可删除、算法透明、操作日志留存、安全评估备案、访问权限管控、数据加密传输、数据加密存储、安全漏洞响应、隐私政策合规、第三方接口安全管控、紧急中断机制。
> 我们可以提供逐项对照清单，每一条有功能截图和配置文档对应。

**Q：等保【注1】做了吗？**
> woclaw 平台架构设计完全符合等保 2.0 三级要求（技术层面）。等保认证需要结合客户的部署环境联合申报，我们可以提供全套配合材料和对接支持，帮助客户通过等保三级。

**Q：数据在客户服务器，你们有没有后门？**
> woclaw 是私有化部署模式，安装包部署到客户服务器后，产品代码完全在客户环境内运行。我们没有任何远程访问通道，也没有数据回传机制。如果客户有要求，可以断网运行（离线模式），整个系统不需要连接互联网即可工作。

## 卡片二：国产适配 · 15+

### PPT 数字含义
**15+** = 已接入并稳定运行的国产大模型数量超过 15 个，且随时可在后台切换，不需要重新部署。

### 说辞
> "国产适配，听起来简单，实际上坑非常多。
> 很多企业用了大厂的 AI 工具，发现一旦那家大厂的模型出问题或者涨价，自己完全没有选择权——因为换模型就等于换平台，成本不可接受。
> woclaw 的思路是：**模型是插件，不是底座。** 今天用 Kimi，明天换通义，后天用豆包，后台切一下，业务不用改动。
> 支持的主流国产模型超过 15 个，包括 Kimi、通义千问、豆包、硅基流动、智谱 GLM、百川、MiniMax 等。
> IM 打通这个更重要——飞书、钉钉、企业微信、微信、QQ，五大平台全部原生支持，一套 Agent 配置，五个渠道同时跑。"

### 深问备答

**Q：15+ 模型是怎么集成的，每个都自己写接口吗？**
> 我们使用统一的模型抽象层（Model Adapter 架构）。所有模型通过标准 OpenAI 兼容接口或我们自研的适配器接入，新增一个模型通常只需要写一个适配器文件（约 200 行代码），不影响上层 Agent 逻辑。
> 目前已验证的模型：DeepSeek V3/R1、Kimi（Moonshot）、通义千问（Qwen 全系列）、豆包（Doubao）、硅基流动（SiliconFlow）、智谱 GLM-4、百川 Baichuan、MiniMax、零一万物（Yi）等。

**Q：微信个人号能接入吗？这个有封号风险吧？**
> 个人微信接入我们采用腾讯官方开放的接口方案（2026 年 3 月微信团队官方发布了 OpenClaw 插件），非逆向协议，不存在封号风险。企业微信是完全官方 API，无任何风险。
> 如果客户对个人微信有顾虑，建议优先使用企业微信方案，稳定性更高。

**Q：模型切换后，之前的对话历史、知识库还在吗？**
> 全部保留。知识库、对话历史、Agent 配置存储在本地数据库中，与模型完全解耦。切换模型就像换了一个"大脑"，但"记忆"和"工具"都不变。

## 卡片三：开放扩展 · 13000+

### PPT 数字含义
**13000+** = ClawHub 技能插件市场中可用的插件数量，覆盖各行各业的自动化工具。

### 说辞
> "一个 AI Agent 能不能用起来，最终取决于它能干什么活——也就是技能和工具有多少。
> woclaw 基于开源生态，接入了 ClawHub 技能市场，13000 多个插件，覆盖从搜索、文件处理、ERP 对接到行业垂直工具的方方面面。
> 更重要的是：**我们不锁定**。今天用 woclaw，如果明天有更好的方案，模型可以换、IM 可以换、甚至底层平台可以迁移——数据是你的，工具是你的，选择权永远在你手里。
> 定制需求我们可以做，时间上按**周级**响应，不是排半年队。"

### 深问备答

**Q：13000 个插件都能用吗，质量怎么保证？**
> ClawHub 是社区插件市场，插件质量参差不齐是客观存在的问题。实际上企业场景常用的核心插件约 200-500 个（搜索、文件、数据库、ERP 对接、通知推送等），这部分我们都经过内部验证。
> 对于企业定制插件，我们提供标准 Plugin SDK，开发一个新插件平均 1-3 天，我们团队可以承接定制开发。

**Q："周级交付"是什么意思，能承诺吗？**
> 针对标准化需求（比如对接某个 ERP 系统、增加一个审批流程节点），我们的交付周期是 5-10 个工作日。
> 复杂定制（比如全新行业垂直 Agent 从零搭建）视需求复杂度，通常 2-4 周。
> 这里的"周级"是针对标准化需求，我们不夸大，会在签约时明确 SLA。

## 进入第二页时说

---

## 四象限图讲解

### 坐标轴解释
> "横轴是**开放灵活度**：越靠右，模型可以自由切换、插件可以自由扩展、系统可以自由集成——不被任何一家厂商绑定。
> 纵轴是**安全稳定性**：越靠上，数据越安全（在自己手里），系统越稳定（有专业团队维护），合规越有保障。"

### 大厂 Claw（左上）

> "字节的 Coze、腾讯的元器、阿里的钉钉 AI——这些大厂产品安全性不低，背后有专业运维团队、有等保认证、系统稳定。
> 但问题在开放性：**数据在大厂服务器**（即使私有化版本推出时间不长，企业信任度有限），**模型绑定**大厂自家（腾讯元器只能用混元，字节主推豆包），更重要的是——**IM 只打通自己家的**，腾讯不会帮你接飞书，字节不会帮你深度支持企业微信。
> 所以大厂产品落在：安全高、开放低——**安全孤岛**。"

### 深问备答

**Q：字节 Coze 不是开源了吗，也支持私有化部署了？**
> 是的，Coze Studio 于 2025 年 7 月以 Apache 2.0 开源，理论上可以私有化部署。但有几个问题：
> 1. **开源时间短**（不到 1 年），社区生态、稳定性、安全审计都还在建立中；
> 2. **国内 IM 适配**：Coze 在飞书、微信生态做了一些，但钉钉、企业微信是竞争关系，深度支持存疑；
> 3. **无专业团队本地支持**：开源自部署意味着出了问题自己解决，字节不会帮你维护私有化版本；
> 4. **woclaw 基于 OpenClaw**，已有完整的中国生态适配和专业团队，不是从零开始。

**Q：腾讯元器安全性怎么样？**
> 腾讯在安全合规方面确实有积累（等保、ISO27001），在腾讯自己的云上安全性没问题。
> 但核心问题是：**数据主权**。你的数据在腾讯的服务器里，腾讯作为平台方有数据访问能力——这对金融、医疗、政务等敏感行业是硬伤。
> 另外，混元模型锁定问题：2026 年大模型竞争白热化，半年内模型能力变化很大，被锁在一个模型上风险很高。

---

### OpenClaw（右下）

> "OpenClaw 是这个领域的开源之王——247,000+ GitHub Stars，MIT 协议，完全开源，支持 200+ 模型，开放性毫无疑问是最高的。
> 问题在安全稳定性这一列：
> 第一，**无专业维护团队**——原作者 2026 年 2 月加入 OpenAI 后，项目交给开源基金会，靠社区驱动。出了问题，你去 GitHub 发 Issue，没有人给你 SLA；
> 第二，**没有等保合规认证**——框架本身没有，需要企业自己做，成本很高；
> 第三，**中国 IM 适配靠社区**——虽然有社区插件，但维护稳定性不保证，企微/微信 API 一旦变动，社区插件可能一夜失效；
> 所以 OpenClaw 落在：开放高、安全低——**开放有险**。"

### 深问备答

**Q：那 woclaw 不就是基于 OpenClaw 的吗，OpenClaw 有的问题 woclaw 也有？**
> 非常好的问题，这正是 woclaw 的核心价值所在。
> **woclaw = OpenClaw 开源底座 + 中国企业级商业化封装**。
> 具体来说，我们做了四件 OpenClaw 没有的事：
> 1. **专业运维团队 + 本地 SLA**：7x24 响应，出了问题有人兜底，不是去 GitHub 发 Issue；
> 2. **中国 IM 深度适配**：飞书/钉钉/企微/微信/QQ 五大平台原生支持，稳定维护，API 变动我们第一时间更新；
> 3. **合规认证支持**：协助客户申请等保，提供全套合规材料；
> 4. **国产模型深度优化**：针对国内网络环境、模型 API 特性做了大量优化，直接可用，不需要客户自己踩坑。

**Q：OpenClaw 社区那么活跃，为什么不直接用开源版？**
> 可以类比：Linux 是开源的，但企业为什么买 Red Hat？因为开源本身不解决**支持、认证、稳定性**三个企业级问题。
> woclaw 就是 AI Agent 领域的"Red Hat"——底层是开源的，上面是企业可以信任的产品。

## 右侧优势列表逐条讲解

### 01 数据100%本地

> "这一条是一票否决项。金融、医疗、政务、制造业客户，第一个问题永远是'数据在哪里'。
> woclaw 的答案是：在你的机房，在你的服务器，我们的代码跑在你那边，我们的服务器里没有你的任何数据。
> 工信部合规 12/12——监管来了不用怕。"

**名词解释：**
- **工信部合规 12/12**：工业和信息化部对企业使用 AI 服务的 12 项安全要求，woclaw 每一项都有对应实现

### 深问备答

**Q：私有化部署后，更新怎么办，你们能推送代码吗？**
> 更新通过我们提供的升级包（离线包）进行，由客户的运维团队自主操作安装。我们不会也不能主动往客户服务器推送任何代码。版本升级是"拉"模式（客户主动拉取），不是"推"模式。

**Q：离线模式下，模型调用怎么办，还是要联网吧？**
> 分两种情况：
> 1. 如果使用云端大模型 API（如 Kimi、通义），需要服务器能访问对应的 API 接口，但这是单向出站请求，不需要外部访问你的服务器；
> 2. 如果需要完全断网，可以在内网部署本地模型（如基于 Ollama 运行的 Qwen、DeepSeek 本地版），整个系统完全不出网。

---

### 02 专业团队保障

> "这一条解决的是 OpenClaw 开源版的最大痛点。
> 开源很好，但开源没有人跟你签 SLA——出了问题，你自己去 Stack Overflow 找答案。
> woclaw 背后有专职团队，7x24 响应，有版本路线图，有安全漏洞响应机制，有升级保障。
> 企业用 AI 工具，最怕的不是贵，而是出了事没人负责。"

**名词解释：**
- **SLA（Service Level Agreement，服务等级协议）**：明确规定服务响应时间、可用率、故障恢复时间的合同约定，比如"故障 4 小时内响应，24 小时内恢复"

### 深问备答

**Q：你们团队多少人？能支撑多少客户？**
> 核心技术团队当前规模，可以支持同时 20-50 家企业的日常运维和支持需求。规模化扩张会同步扩充支持人员。具体 SLA 条款（响应时间、处理时限、可用性保证）在合同中明确约定。

**Q：如果你们公司倒闭了怎么办，代码还能跑吗？**
> woclaw 底层是 OpenClaw 开源代码，MIT 许可证，永久可用。即使极端情况下我们不再提供服务，客户已部署的系统仍可继续运行，数据和代码都在客户手里，不存在"服务商跑路数据丢失"的风险。这也是基于开源而非私有代码构建的核心优势。

---

### 03 15+ 模型零绑定

> "大模型领域迭代极快，今年最强的模型明年可能被超越。
> 被锁定在一个模型上，意味着你无法享受技术进步的红利。
> woclaw 的设计哲学是：**模型是工具，不是基础设施**。随时可换，成本为零。
> 15 个国产模型已接入，后续新出的模型，我们会持续跟进。"

**名词解释：**
- **零绑定（Zero Lock-in）**：系统不对任何单一供应商产生依赖，可以随时替换核心组件（模型、数据库、IM 渠道等）
- **OpenAI 兼容接口**：大多数国产大模型提供与 OpenAI 相同格式的 API，便于统一接入

### 深问备答

**Q：切换模型会不会导致 Agent 行为变化，效果变差？**
> 不同模型的能力有差异，切换模型后 Agent 的回答质量、风格可能有变化，这是客观事实。
> 我们提供模型评估工具，帮助客户在切换前做 A/B 测试，确认效果后再正式切换。对于关键业务场景，建议做回归测试。

**Q：本地部署的模型（Ollama）能用吗，性能怎么样？**
> 完全支持。基于 Ollama 可以在客户服务器上运行 Qwen-7B、DeepSeek-7B 等小尺寸模型，性能与服务器 GPU 配置直接相关。
> 推荐配置：A100 80G 运行 70B 模型效果较好；RTX 4090 运行 14B 模型是性价比方案。
> 完全本地模型的优势是：不需要任何出网访问，适合最高安全级别场景。

---

### 04 全平台IM打通

> "中国企业的沟通生态是碎片化的——不同企业用不同的 IM，甚至同一家企业内部多个 IM 并存。
> woclaw 支持飞书、钉钉、企业微信、微信、QQ，**一套 Agent 配置，五个渠道同时服务**，员工在哪里，AI 就在哪里，不需要员工改变工作习惯。"

**名词解释：**
- **IM（Instant Messaging，即时通讯）**：飞书、钉钉、企微等企业沟通工具
- **原生支持**：不是通过 webhook 勉强转发，而是深度集成了各平台的消息格式、附件、卡片消息、事件推送等能力

### 深问备答

**Q：微信个人号接入是官方支持还是破解？**
> 官方支持。2026 年 3 月，腾讯微信团队官方发布了 OpenClaw 的微信插件（`@tencent-weixin/openclaw-weixin`），属于腾讯官方认可的接入方式，无封号风险。企业微信更是完全基于官方开放平台 API 构建。

**Q：如果客户只用钉钉，不用其他 IM，多余的集成会不会带来安全风险？**
> 各 IM 渠道是独立模块，不使用的渠道可以在后台完全禁用，不会启动也不会产生任何流量。权限模型是最小化原则，不启用的功能不会加载。

### 05 13000+ 开放生态

> "光有对话能力不够，AI Agent 的价值在于能帮企业干实事——搜资料、写报告、对接 ERP、自动审批、推送通知……
> ClawHub 的 13000+ 插件覆盖了大部分常见场景，开箱即用。
> 需要定制的，我们周级响应，不是排队等半年。"

**名词解释：**
- **AI Agent（智能体）**：能够自主执行多步骤任务的 AI 程序，不只是聊天，而是能主动调用工具、浏览网页、操作系统、调用 API 来完成复杂任务
- **ClawHub**：woclaw/OpenClaw 的技能插件市场，类似 App Store，开发者可上传插件，用户一键安装

### 深问备答

**Q：13000 个插件怎么选，有没有审核机制？**
> ClawHub 对上架插件有基础安全审核（代码扫描、权限声明审查）。企业用户建议从"官方认证"和"高下载量"插件中选择。
> 对于企业生产环境，我们建议客户维护一个"内部白名单"，只允许经过客户 IT 审核的插件运行，woclaw 后台支持插件白名单管理。

**Q：能对接我们公司内部系统（ERP/OA/CRM）吗？**
> 可以。只要内部系统有 HTTP API 或数据库接口，就可以对接。常见的 SAP、金蝶、用友、钉钉 OA、企业微信 OA 我们都有现成的适配器。定制对接周期一般 1-2 周，视接口复杂度而定。

## woclaw vs 字节 Coze

### woclaw 优势
1. **IM 全覆盖**：Coze 深度优化飞书（字节自用），钉钉/企微属于竞争对手，官方支持意愿和深度有限；woclaw 五大平台无差别深度支持
2. **本地化更成熟**：Coze Studio 开源时间短（2025年7月），中国企业场景的坑还没踩完；woclaw 专为中国企业设计，已有落地案例
3. **专业本地支持**：Coze 私有化版本出了问题，字节不会派人上门；woclaw 有本地团队
4. **无平台依赖**：用 Coze 等于依赖字节的产品路线图，字节说砍功能就砍；woclaw 独立运营

### woclaw 相对劣势
1. **品牌知名度**：Coze 已是国内知名度最高的 AI Agent 平台，获客容易
2. **SaaS 快速体验**：Coze 云端版免费注册即用，woclaw 需要部署
