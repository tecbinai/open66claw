## 三、交互建议（按优先级排序）

### 🔴 P0 — 必须补齐（不补则功能残缺）

#### 1. 能力卡点击→快速切换

**现状问题**：Figma 只画了能力卡的静态展示，没有交互态。
**建议**：点击能力卡展开右侧联动 — 右侧服务商列表自动筛选到"能提供此能力的 Provider"，高亮当前使用的模型。

```
点击「聊天」卡片
  → 右侧：只显示支持 text 能力的 Provider（灰掉其他的）
  → 当前使用的模型高亮标记 ✓
  → 点击另一个 Provider 的模型 → 直接切换
```

**这比现有的"展开面板"更自然** — 因为 Figma 已经是双栏布局了，左侧选能力、右侧选模型，天然的联动关系。

#### 2. Provider 配置向导

**现状问题**：Figma 没有画配置向导弹窗（guide→apikey→detecting→result 四步）。
**建议**：保留现有的 4 步弹窗，但视觉风格跟随 Figma 的浅色主题。配置流程是核心路径，不能省。

#### 3. Provider 管理弹窗

**现状问题**：Figma 的"查看配置"按钮点击后进入什么界面没画。
**建议**：保留现有的管理弹窗（脱敏 Key + 添加模型 + 测试 + 删除），视觉调整。

#### 4. 空状态引导

**现状问题**：Figma 画的是有数据的状态，没有画"全空"时怎么办。
**建议**：全空时，左侧能力卡全灰 + 右侧显示"开始配置"引导，推荐 Kimi Code 一键上手。

---

### 🟡 P1 — 应该补齐（提升体验）

#### 5. 左右联动交互

**核心建议：这是 Figma 双栏布局的最大价值，但 Figma 没有画交互细节。**

```
【联动规则】

状态 A：默认态
  左侧：6 张能力卡，显示状态
  右侧：全部 Provider 列表

状态 B：选中某个能力（如"聊天"）
  左侧：「聊天」卡片高亮选中态
  右侧：
    ├── 支持聊天的 Provider → 正常显示 + 当前模型 ✓ 标记
    ├── 不支持聊天的 Provider → 灰显 + 标注"不支持此能力"
    └── 顶部出现浮条："当前聊天模型: DeepSeek V3 (deepseek)"

状态 C：在右侧点击某个 Provider 的模型
  → 切换绑定 → 左侧卡片实时更新 → Toast "已切换到 xxx"
```

#### 6. Provider 卡片的多态

Figma 画了 3 种卡片风格，建议明确定义：

| 态         | 视觉          | 按钮                      | 说明                  |
| ---------- | ------------- | ------------------------- | --------------------- |
| 推荐未配置 | 渐变色高亮    | 「一键开通」              | 顶部推荐区的 3 张     |
| 已配置     | 蓝色/紫色渐变 | 「查看配置」/「查看模型」 | 已有 API Key          |
| 未配置     | 白色普通      | 「配置」                  | 底部列表              |
| 健康异常   | 红色边框      | 「管理」                  | API Key 过期/余额不足 |

#### 7. 错误状态 & Loading

**建议补充的状态：**

- Loading 骨架屏（首次进入页面）
- Gateway 断连态（灰色蒙层 + "服务未连接"）
- 单个 Provider 检测失败态（红色边框 + 错误提示）

---

### 🟢 P2 — 可以简化（降低复杂度）

#### 8. 拖拽排序 → 简化或隐藏

**建议**：拖拽排序是高级功能，小白不需要。

- 默认隐藏，放到"高级设置"入口
- 或者改为简单的"上移/下移"按钮

#### 9. 分组折叠 → 用 Tab/标签页替代

**现状**：5 个分组折叠/展开，操作繁琐。
**Figma 方案**：顶部推荐 + 网格列表，没有分组折叠。
**建议**：右侧顶部加一排 Tab 筛选：「推荐」「国内」「国际」「本地」，切换显示不同分组。比折叠更直觉。

#### 10. 能力等级（强/中/弱）→ 轻量化

**现状**：badge + 警告框，视觉很重。
**建议**：简化为模型名旁边的小标签，不要用警告框打断用户。

## 五、关键交互建议总结

### 建议 1：左右联动是灵魂

Figma 画了双栏布局但没有画联动交互。**这是改版最大的机会**。

现有代码：能力卡和 Provider 列表各管各的，用户要在脑子里做映射。
新设计：左侧选能力 → 右侧自动筛选+高亮 → 直觉式操作。

### 建议 2：三种用户路径都要顺畅

| 用户         | 路径                                        | 触发点           |
| ------------ | ------------------------------------------- | ---------------- |
| **新手**     | 进入 → 看到推荐 → 一键开通 Kimi → 能聊天了  | 推荐区"一键开通" |
| **普通用户** | 想换模型 → 点聊天卡 → 右侧看到可选 → 点切换 | 左侧能力卡       |
| **高级用户** | 管理多个 Key → 查看健康状态 → 调整优先级    | 右侧"管理"按钮   |

### 建议 3：去掉让小白困惑的术语

| 现在的术语   | 建议改为                    |
| ------------ | --------------------------- |
| Provider     | 服务商 / AI 服务            |
| API Key      | 密钥 / 账号密钥             |
| Capability   | 能力                        |
| Embedding    | 记忆（向量嵌入细节隐藏）    |
| toolCall     | 不显示（系统内部能力）      |
| Priority     | 优先顺序（藏到高级设置）    |
| Failover     | 自动切换（不解释机制）      |
| Probe/Detect | 检测（"正在检测您的密钥…"） |

### 建议 4：Provider 卡片内嵌能力矩阵

Figma 的 Provider 卡片只显示名称+能力标签。建议增加：

```
┌────────────────────────────────────┐
│ 🔥 Kimi Code                   免费 │
│ 聊天 · 编程 · 看图                    │
│ 新手首选，注册即用                     │
│                                     │
│ 使用中的能力:                         │ ← 只在已配置卡片显示
│ 💬 聊天  kimi-for-coding    ✓       │
│ 💻 编程  kimi-k2           ✓       │
│                                     │
│            [查看模型]  [管理]         │
└────────────────────────────────────┘
```

这样用户能直接看到"这个服务商正在给我提供什么"。

### 建议 5：配置成功后的"庆祝感"

Figma 没画配置完成态。建议：

```
┌────────────────────────────────────┐
│  🎉 配置成功！                       │
│                                     │
│  已自动开通以下能力:                   │
│  ✅ 聊天  → deepseek-chat           │
│  ✅ 编程  → deepseek-coder          │
│  ✅ 工具  → deepseek-chat           │
│                                     │
│  左侧能力卡已更新，立即试试吧！         │
│                                     │
│              [开始聊天]  [继续配置]    │
└────────────────────────────────────┘
```
