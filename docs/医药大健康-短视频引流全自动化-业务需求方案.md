## 二、M1 爆款素材采集

### 2.1 业务目的

解决**"发什么能火"**的问题。快速找到行业内正在爆的视频和图文，提取选题方向、内容结构、热门标签，作为内容生产的输入源。

### 2.2 业务流程

```
维护关键词库 → 三平台并发搜索 → 按爆款条件筛选 → 列表预览
    → 批量下载(视频/图文/文案/封面) → 入库分类打标签 → 爆款拆解分析
```

### 2.3 功能需求

| 编号 | 功能 | 说明 |
|------|------|------|
| 1.1 | 关键词库管理 | 维护行业关键词（养生/调理/中医/免疫力/睡眠/食疗…），支持增删改查 |
| 1.2 | 三平台统一搜索 | 输入关键词，同时搜索抖音、快手、小红书，结果统一展示 |
| 1.3 | 爆款条件筛选 | 支持按点赞量、评论量、收藏量、发布时间、博主粉丝范围等条件过滤 |
| 1.4 | 结果列表预览 | 缩略图 + 标题 + 核心数据（赞/评/藏），支持批量勾选 |
| 1.5 | 一键批量下载 | 去水印下载视频/图文/文案/封面/BGM 信息 |
| 1.6 | 素材库管理 | 按平台、话题、类型、热度自动分类打标签，支持搜索和筛选 |
| 1.7 | 爆款拆解 | 记录爆款的开头钩子、内容结构、话术风格、标签组合，形成可复用的选题卡 |

### 2.4 爆款筛选条件参考

| 条件 | 抖音 | 快手 | 小红书 |
|------|------|------|--------|
| 点赞量 | ≥3,000 | ≥2,000 | ≥1,000 |
| 评论量 | ≥500 | ≥300 | ≥200 |
| 收藏量 | — | — | ≥500（核心指标）|
| 发布时间 | 最近 72 小时 | 最近 72 小时 | 最近 7 天（长尾效应）|
| 点赞率 | ≥3% | ≥5% | — |
| 博主粉丝范围 | 可自定义过滤 | 可自定义过滤 | 可自定义过滤 |
| 关键词命中 | 标题/文案/标签 | 标题/文案/标签 | 标题/文案/标签 |

> 所有阈值均支持用户自定义调整。

## 三、M2 素人账号采集

### 3.1 业务目的

找到**真实的普通人账号**，下载其主页信息及历史作品。用途有二：一是作为二次创作的素材来源（真实感强），二是学习素人的表达风格用于养号。

### 3.2 业务流程

```
设定素人筛选条件 → 平台搜索/发现 → 系统自动判定是否素人
    → 符合条件进入列表 → 一键下载主页信息 + 全部作品 → 归档入素材库
```

### 3.3 素人账号判定规则

| 维度 | 符合条件 | 排除条件 |
|------|----------|----------|
| 粉丝量 | 100 ~ 5,000 | <100（僵尸号）或 >5,000 |
| 认证状态 | 无蓝V / 无黄V | 有任何认证标识 |
| 商业痕迹 | 无橱窗 / 无商品链接 / 无星图标识 | 有任何商业化功能 |
| 内容风格 | 生活化、非专业制作 | 明显的包装感或专业剪辑 |
| 发布频率 | 不规律 | 固定日更（像营销号）|
| 账号年龄 | ≥3 个月 | 近期新注册 |

### 3.4 采集内容

| 采集项 | 说明 |
|--------|------|
| 主页信息 | 头像、昵称、简介、粉丝数、获赞数、IP 属地 |
| 全部作品 | 视频文件 + 封面 + 文案 + 标签 + 各项数据（赞/评/藏/转）|
| 评论样本（可选）| 该账号在其他视频下发过的评论，用于学习真实评论风格 |

## 四、M3 内容自动化生产

### 4.1 业务目的

解决**"怎么批量做出能火的内容"**的问题。基于爆款素材、专业知识库、素人素材，系统化生产短视频和图文内容，并生成多个差异化版本供矩阵账号使用。

### 4.2 三大输入源

```
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  ① 爆款素材库  │   │ ② 专业知识库   │   │  ③ 素人素材库  │
│               │   │               │   │               │
│ · 热门选题     │   │ · 医药专业知识 │   │ · 素人作品     │
│ · 爆款结构模板 │   │ · 养生食疗方   │   │ · 真实表达风格 │
│ · 热门BGM/标签│   │ · 中医科普     │   │ · 生活化场景   │
│               │   │ · 合规话术库   │   │ · 口播节奏     │
└───────┬───────┘   └───────┬───────┘   └───────┬───────┘
        │                   │                   │
        └───────────┬───────┘───────────┬───────┘
                    ▼                   ▼
            ┌───────────────────────────────┐
            │        内容生产引擎            │
            │  选题 → 脚本 → 素材 → 成片    │
            └───────────────────────────────┘
```

### 4.3 专业知识库

这是医药行业区别于其他行业的**核心壁垒**。

#### 知识库内容结构

| 分类 | 内容 | 来源 | 示例 |
|------|------|------|------|
| 养生科普 | 日常调理、食疗方、节气养生 | 中医教材 / 公开文献 | "春季养肝三个习惯" |
| 成分百科 | 草本植物、食材功效、营养成分 | 药典 / 营养学教材 | "黄芪的 5 个日常用法" |
| 生活方式 | 作息、运动、饮食、情绪管理 | 健康指南 / WHO 建议 | "失眠人群的睡前 30 分钟" |
| 常见误区 | 大众认知错误、伪科学辟谣 | 权威医学机构发布 | "这 3 个养胃方法其实是错的" |
| 时令热点 | 换季、流感季、节气对应内容 | 时事 + 专业结合 | "倒春寒，这类人要特别注意" |
| 合规话术 | 安全用词替换表、禁用词清单 | 广告法 / 平台规则 | "调理"替代"治疗" |

#### 知识库管理要求

| 需求 | 说明 |
|------|------|
| 标签化 | 每条知识打标签（症状 / 人群 / 季节 / 场景），方便检索匹配 |
| 合规审核 | 入库前审核，确保不含违规表述 |
| 定期更新 | 跟随季节、热点、平台规则变化更新 |
| 权威溯源 | 每条知识标注来源出处，可追溯 |

### 4.4 爆款模板库

从采集到的爆款素材中拆解出**可复用的内容结构**。

#### 视频模板

| 模板类型 | 结构 | 示例标题 |
|----------|------|----------|
| 恐惧开头型 | 抛出问题 → 放大痛点 → 给出方案 → 引导关注 | "还在这样喝水？你的肾在求救" |
| 反常识型 | 颠覆认知 → 解释原因 → 正确做法 | "红枣补血？大错特错" |
| 清单型 | 数字标题 → 逐条讲解 → 引导收藏 | "中医推荐的 5 个泡脚方" |
| 故事型 | 个人经历 → 遇到问题 → 如何解决 → 现状 | "我妈失眠 10 年，后来…" |
| 对比型 | 错误做法 vs 正确做法 | "90% 的人枸杞都吃错了" |
| 热点借势型 | 热点事件 → 关联健康话题 → 专业解读 | "XX 事件背后的健康真相" |

#### 图文模板（小红书为主）

| 模板类型 | 结构 | 示例 |
|----------|------|------|
| 教程卡片 | 封面标题图 + 3~6 张步骤图 + 文案 | "祛湿茶配方｜每天一杯" |
| 知识图谱 | 一张信息量大的科普长图 | "9 种体质自测表" |
| 合集种草 | 多方法/多好物合集 | "打工人养胃好物合集" |
| 问答型 | 模拟用户提问 → 专业解答 | "经常熬夜怎么补救？" |

#### 模板管理要求

- 按平台分类（抖音偏视频、小红书偏图文、快手偏生活化）
- 标注适用人群（女性 / 中老年 / 上班族 / 宝妈）
- 记录使用该模板的历史作品数据，持续优化
- 效果持续下滑的模板降权或淘汰

### 4.5 内容生产完整流程

```
S1 选题生成
    爆款库热门选题 + 知识库匹配 + 时令热点 → 输出"选题卡"
         ↓
S2 脚本生成
    选题卡 + 爆款模板结构 + 知识库内容 → 输出脚本初稿
         ↓
S3 合规审核
    脚本过敏感词库自动扫描 → 标红违规词 + 推荐替换 → 人工确认
         ↓
S4 素材匹配
    根据脚本内容匹配素材库中的视频片段 / 图片 / BGM
         ↓
S5 成片生成
    ├─ 视频：脚本 + 配音 + 画面素材 + 字幕 + BGM → 合成
    └─ 图文：文案 + 排版模板 + 配图 → 生成图文卡片
         ↓
S6 差异化处理
    同一内容生成多个版本，供矩阵不同账号使用
         ↓
S7 入库排期
    成片进入"待发布内容库" → 按账号 / 平台 / 时段分配发布计划
         ↓
S8 数据回收
    发布后追踪数据 → 反馈到选题库和模板库 → 优化下一轮生产
```

#### S1 选题生成规则

| 选题来源 | 权重 | 逻辑 |
|----------|------|------|
| 爆款库近 3 天热门选题 | 40% | 已验证有流量，成功率高 |
| 专业知识库匹配 | 30% | 有专业深度，建立信任 |
| 时令 / 节气 / 热点 | 20% | 借势流量，时效性强 |
| 竞品账号分析 | 10% | 参考同行表现 |

**输出物**：选题卡（标题方向 + 目标人群 + 参考爆款 + 对应知识点 + 推荐模板）

#### S2 脚本要素

| 要素 | 说明 |
|------|------|
| 开头钩子（前 3 秒）| 从爆款模板中提取，决定用户是否继续看 |
| 正文内容 | 从知识库提取专业内容，用口语化表达重写 |
| 结尾引导 | 关注 / 收藏 / 评论引导语 |
| 字幕文案 | 配合画面的文字版本 |
| 标签 / 话题 | 从爆款库提取高流量标签组合 |
| 时长控制 | 抖音 15~60 秒、小红书图文 6~9 张图 |

#### S3 合规审核 — 敏感词替换表

| 禁用词 | 安全替换 |
|--------|----------|
| 治疗 / 治愈 | 调理 / 改善 |
| 药 / 保健品 | 好物 / 好方法 |
| 疗效显著 | 感觉不错 / 舒服多了 |
| 患者 / 病人 | 朋友 / 家人 |
| 根治 / 痊愈 | 好转 / 有变化 |
| 降压 / 降糖 | 注意饮食习惯 |
| 处方 / 医嘱 | 建议 / 经验分享 |

#### S6 差异化处理维度

同一选题/脚本生成多个版本，避免平台判定搬运或重复内容：

| 差异化维度 | 做法 |
|-----------|------|
| 封面 | 换文案 / 配色 / 排版 |
| 开头 | 换不同的钩子句 |
| BGM | 换不同的背景音乐 |
| 配音 | 换音色 / 语速 |
| 画面 | 镜像翻转 / 调色 / 换素材片段 |
| 文案 | 同义改写，保持意思不变 |
| 标签 | 换不同的标签组合 |

### 4.6 内容分发排期

| 维度 | 规则 |
|------|------|
| 发布时段 | 早 7~9 点 / 午 12~14 点 / 晚 19~22 点（流量高峰）|
| 发布频率 | 内容号：每天 1~2 条；引流号：每周 2~3 条 |
| 平台适配 | 同一选题发不同平台时，调整时长 / 格式 / 标签 |
| 矩阵错开 | 同一内容的不同版本，错开至少 2 小时发布 |
| 热点响应 | 突发热点 2 小时内出内容，抢时效 |

**分发示例**：

```
选题 A 生成 5 个版本：
  ├── 版本 1 → 内容号 D（抖音）  早 8:00 发布
  ├── 版本 2 → 内容号 E（抖音）  午 12:30 发布
  ├── 版本 3 → 内容号 F（快手）  晚 19:00 发布
  ├── 版本 4 → 小红书号 G       早 9:00 发布（图文版）
  └── 版本 5 → 小红书号 H       晚 20:00 发布（图文版）
```

## 五、M4 评论区截留引流

### 5.1 业务目的

在他人热门视频评论区**抢占靠前位置**，通过有价值的评论吸引用户点击头像 → 回访主页 → 进入私域。这是整个引流体系的**核心动作**。

### 5.2 业务流程

```
┌──────────────────────────────────────────────────────────────────┐
│  准备阶段                                                         │
│                                                                   │
│  目标视频筛选              话术库维护             账号池管理        │
│  (在哪些视频下评论)        (说什么)              (用哪些号)        │
└──────┬──────────────────────┬───────────────────────┬────────────┘
       │                      │                       │
       ▼                      ▼                       ▼
┌──────────────────────────────────────────────────────────────────┐
│  执行阶段                                                         │
│                                                                   │
│  生成任务（视频 + 话术 + 账号 三者匹配）                            │
│       ↓                                                           │
│  自动执行：观看视频 → 浏览评论区 → 发布评论                         │
│       ↓                                                           │
│  互顶机制：矩阵内小号给评论点赞，帮助评论上热评前排                  │
└──────┬───────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  监控阶段                                                         │
│                                                                   │
│  评论状态监控（是否被删 / 是否进入前排）                             │
│  引流效果监控（主页访问量 / 新增粉丝 / 新增私信）                    │
│  数据反馈 → 优化话术、目标视频选择策略                               │
└──────────────────────────────────────────────────────────────────┘
```

### 5.3 目标视频筛选规则

| 维度 | 标准 |
|------|------|
| 行业相关性 | 视频内容与医药 / 健康 / 养生直接相关 |
| 评论区活跃度 | 评论量 ≥200，且有真实讨论 |
| 博主粉丝量 | 10 万 ~ 100 万（中腰部最佳）|
| 发布时间 | **优先 30 分钟内的新视频**（抢前排黄金窗口）|
| 数据趋势 | 数据正在上升中的视频 |

### 5.4 截留时机

| 时机 | 视频发布后 | 效果 | 说明 |
|------|-----------|------|------|
| 黄金窗口 | 0~30 分钟 | ★★★★★ | 评论少，早期评论容易被推到前排 |
| 优质窗口 | 30 分钟~2 小时 | ★★★★ | 配合小号互顶可以上热评 |
| 常规窗口 | 2~6 小时 | ★★★ | 需要靠高质量"神评论"突围 |
| 长尾窗口 | 6 小时以上 | ★★ | 仅针对持续爆火的视频 |

### 5.5 话术分层体系

| 层级 | 名称 | 占比 | 目的 | 示例 |
|------|------|------|------|------|
| L1 | 纯互动 | 60% | 养号权重，维持账号活跃 | "太对了，深有同感" |
| L2 | 价值补充 | 20% | 展示专业度，吸引主动关注 | "补充一下，XX 情况还可以…" |
| L3 | 软引导 | 15% | 引发好奇，促使回访主页或私信 | "之前也有这困扰，后来找到了方法" |
| L4 | 钩子型 | 5% | 直接收集意向用户（慎用）| "整理了一份 XX 指南，需要的扣 1" |

**话术管理要求**：

- 每层级储备 ≥30 条话术，支持增删改查
- 支持变量替换（随机 emoji、语气词、适当口语化）增加自然感
- 同一视频下不出现重复话术
- 同一账号连续评论不用相似句式
- 话术长度随机变化（5~50 字）

### 5.6 执行节奏

| 参数 | 抖音 | 快手 | 小红书 |
|------|------|------|--------|
| 单号日评论上限 | 15~20 条 | 20~30 条 | 10~15 条 |
| 评论间隔 | 1~3 分钟随机 | 1~3 分钟随机 | 2~4 分钟随机 |
| 活跃时段 | 8:00~23:00 | 8:00~23:00 | 8:00~23:00 |
| 休息机制 | 每 3~5 条休息 5~15 分钟 | 同左 | 同左 |
| 异常熔断 | 检测到验证码或弹窗提示时立即暂停 | 同左 | 同左 |

## 六、M5 点赞引流

### 6.1 业务目的

通过批量点赞目标用户的作品，触发平台通知 → 用户好奇回访主页 → 被主页内容吸引 → 关注或私信。

### 6.2 业务流程

```
筛选目标用户（看过/评论过行业视频的人）
    ↓
对每人连续点赞 3~5 条作品（增加通知频率和关注度）
    ↓
用户收到多条点赞通知 → 好奇点击头像 → 回访主页
    ↓
主页简介 + 置顶内容承接 → 引导关注或私信
```

### 6.3 运营参数

| 项目 | 参数 |
|------|------|
| 单号日操作量 | 200~500 次 |
| 预期回访率 | 1~3% |
| 风险等级 | **低**（点赞是平台鼓励的正常行为）|
| 核心前提 | 主页必须优化好——简介有钩子、置顶有价值内容 |

---

## 七、M6 关注引流

### 7.1 业务目的

批量关注目标用户，利用社交礼仪性回关心理，建立粉丝关系后再通过私信或内容做进一步转化。

### 7.2 业务流程

```
筛选目标用户
    ↓
三步关注法：先点赞 → 再评论 → 再关注（回关率显著提升）
    ↓
等待 24~72 小时
    ↓
已回关 → 进入私信转化流程
未回关 → 定期取关清理（保持关注/粉丝比健康）
```

### 7.3 运营参数

| 项目 | 参数 |
|------|------|
| 单号日操作量 | 50~100 次 |
| 预期回关率 | 三步关注法：10~15%（直接关注仅 3~5%）|
| 风险等级 | **中**（平台有日关注上限，超过会限流）|
| 适用场景 | 冷启动阶段快速建立粉丝基础 |

## 八、M7 私信引流转化

### 8.1 业务目的

对已有互动基础的高意向用户进行一对一私信沟通，引导添加微信进入私域。这是**从公域到私域的关键转化环节**。

### 8.2 触发场景（按优先级排序）

| 优先级 | 触发场景 | 说明 |
|--------|----------|------|
| 最高 | 用户主动私信 | 已有明确意向，必须立即响应 |
| 高 | 用户回复了你的评论 | 有互动基础，趁热跟进 |
| 中 | 用户回关了你 | 有一定兴趣，发送欢迎语 |
| 低 | 用户点赞了你的内容 | 轻度兴趣，可尝试触达 |

### 8.3 四步转化话术

```
第 1 步：打招呼 + 确认需求
  "你好呀，看到你在 XX 视频下的评论，是想了解 XX 方面的信息吗？"

第 2 步：提供价值 + 建立信任
  "我之前也研究过这个问题，有一些经验可以分享…"

第 3 步：引导转移 + 给出理由
  "平台私信经常吞消息，方便的话加个微信沟通更顺畅"

第 4 步：发送联系方式
  （图片形式 / 谐音 / 引导看主页简介 —— 避免直接发微信号被平台屏蔽）
```

### 8.4 运营参数

| 项目 | 新号 | 老号 |
|------|------|------|
| 单号日私信量 | 5~10 条 | 20~30 条 |
| 最佳转化窗口 | 产生互动后 24 小时内 | — |
| 预期回复率 | 10~20% | 15~25% |
| 回复 → 加微信转化率 | 20~40% | 20~40% |
| 风险等级 | **高**（私信限制最严，容易被举报封号）| 中高 |

---

## 九、M8 账号矩阵管理

### 9.1 业务目的

多账号分工协作，分散单号风险，放大整体引流效果。

### 9.2 矩阵架构

```
品牌主号（1 个） ─── 承接流量，展示品牌形象，不直接做引流动作
    │
    ├── 评论截留号（5~10 个）── 在别人热门视频下评论引流
    ├── 内容号（3~5 个）     ── 发布知识科普 / 案例故事
    ├── 互动号（5~10 个）    ── 给截留评论点赞上热评 + 给内容号制造热度
    └── 点赞关注号（5~10 个）── 批量点赞、关注拉新
```

**建议矩阵总规模**：20~50 个账号，轮换使用。

### 9.3 养号生命周期

| 阶段 | 天数 | 操作 | 禁止事项 |
|------|------|------|----------|
| 注册完善 | D1~3 | 完善资料（头像/昵称/简介）、实名认证、绑定手机 | 不发内容、不评论 |
| 冷启动 | D4~7 | 每天刷同领域内容 30~60 分钟，点赞 10~20 个，写 2~3 条真实评论 | 不带任何营销信息 |
| 活跃期 | D8~14 | 发布 1~2 条生活化原创内容，持续日常互动 | 不放联系方式、不打广告 |
| 测试期 | D15~21 | 发布 3~5 条垂直领域内容，观察播放和互动数据 | 数据异常需排查 |
| 正式运营 | D22+ | 按分工执行引流任务 | 遵守各项操作上限 |

### 9.4 封号红线

| 高危行为 | 后果 |
|----------|------|
| 同一设备登录多个账号 | 关联封号 |
| 评论/私信中直接出现微信号、电话、链接 | 删评/禁言/封号 |
| 短时间大量重复操作（评论/点赞/关注）| 限流/降权 |
| 新号直接发广告内容 | 永久封号 |
| 搬运他人内容不做改动 | 限流/下架 |
| 使用模拟器或多开软件 | 设备级封禁 |
| 频繁修改个人简介 | 触发审核 |

### 9.5 降权信号判断

出现以下情况说明账号可能已被降权，需排查处理：

- 视频播放量持续低于 200
- 评论/私信发出后对方看不到
- 搜索自己的昵称找不到账号
- 发布内容长时间显示"审核中"

## 十、M9 私域承接与成交

### 10.1 业务目的

将从公域引流过来的用户沉淀到微信私域，通过持续运营建立信任，最终完成成交转化。

### 10.2 承接方式

| 形式 | 适用场景 |
|------|----------|
| 个人微信号 | 高客单价产品，需要一对一深度信任 |
| 企业微信 | 规模化运营，合规需求高 |
| 微信社群 | 批量运营，需要氛围带动成交 |
| 公众号 / 小程序 | 内容沉淀 + 标准化商品成交 |

### 10.3 私域运营节奏

```
Day 0 ：添加好友，发送欢迎语 + 自我介绍 + 赠送一份"行业资料"建立好感
            ↓
Day 1~3 ：朋友圈展示专业内容（科普、案例、用户反馈）
            ↓
Day 3~7 ：私聊跟进，了解具体需求，提供个性化建议
            ↓
Day 7~14：推荐产品/服务，给出方案
            ↓
Day 14+ ：持续朋友圈运营，定期回访，等待成交时机
```

### 10.4 核心原则

> **公域平台（抖音/快手/小红书）只做知识输出和信任建立，所有产品推荐和成交动作只在私域完成。**

这既是转化效率最高的路径，也是医药行业合规的基本要求。

## 十一、全链路转化漏斗

        │  1~3%
        ▼

        ▼

        ▼

        │  30~50%
        ▼

## 十二、医药行业合规红线

### 12.1 内容合规

| ✅ 可以做 | ❌ 绝对不能做 |
|-----------|--------------|
| 健康科普、养生常识、生活方式建议 | 疾病治疗承诺、功效保证 |
| 个人调理经历分享（第一人称叙述）| 处方药任何形式的推广 |
| 食疗 / 运动 / 作息建议 | 代替就医的暗示 |
| 原创科普内容 | 伪造用户案例、虚假证书 |
| 引用公开的学术研究（注明来源）| 前后效果对比图 |
| "调理 / 改善 / 缓解 / 建议" | "治愈 / 根治 / 100% 有效 / 国家级" |

### 12.2 引流合规

| ✅ 可以做 | ❌ 绝对不能做 |
|-----------|--------------|
| 主页简介间接引导 | 评论/私信中直接放微信号、电话、链接 |
| 评论区输出知识价值、引发互动 | 评论区硬广、群发广告 |
| 通过内容质量吸引自然关注 | 使用机器人刷量（一旦查实永久封号）|

### 12.3 法律底线

| 法规 | 核心要求 |
|------|----------|
| 《广告法》第 16~18 条 | 医疗/药品/保健品广告不得含有功效断言、代言推荐 |
| 《药品管理法》| 处方药严禁面向公众广告；药品广告须经省级药监审批 |
| 《互联网广告管理办法》| 互联网广告须标注"广告"；素人软广同样受监管 |
| 《个人信息保护法》| 不得违规采集用户个人信息 |

**违法后果**：罚款 20~100 万元，严重可追究刑事责任。

### 12.4 合规策略总结

1. **身份定位**：做"健康生活方式分享者"，不做"卖药的"
2. **内容策略**：80% 纯知识科普 + 20% 软性方向引导，不提具体产品
3. **转化分离**：公域只做信任建立，私域才做产品推荐和成交
4. **素材二创**：只借鉴选题和结构，重新组织内容，不搬运不抄袭
5. **合规前置**：所有内容发布前过敏感词库审核，宁可少说不可说错
