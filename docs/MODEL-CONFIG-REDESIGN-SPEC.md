# 模型设置页面改版实施规格

> 基于 Figma 设计稿 + 用户确认的需求，指导 view 层重写。
> 日期：2026-03-09

---

## 一、确认的改版范围

### 改什么

- **布局**：单列 → 双栏（左能力卡 + 右 Provider 网格）
- **配色**：暗色 → 浅色毛玻璃风格（跟随 Figma + 聊天页风格）
- **能力卡**：3×2 网格 → 左侧纵向列表，点击弹出模型选择器
- **Provider 卡片**：上下分区 → 右侧网格，推荐高亮 + 普通白色
- **排序**：拖拽排序 → 点击按钮弹出排序浮动框，框内上下拖拽
- **去掉**：能力等级警告（weak/moderate 红黄提示框），新手引导
- **保留**：健康状态徽章，模型排序能力强的靠前

### 不改什么

- **配置向导弹窗** — 保留现有 4 步流程（guide→apikey→detecting→result）
- **模型选择器弹窗** — 保留现有搜索+筛选+模型列表
- **Provider 管理弹窗** — 保留现有（脱敏 Key + 添加模型 + 测试 + 删除）
- **控制器层** — controllers/model-config.ts 不改
- **Gateway 接口** — 所有 RPC 方法不改
- **后端 Handler** — ui-bridge / provider-registry / config-store / probe 不改

## 三、组件详细规格

### 3.1 左侧 — 我的能力（宽度 ~320px）

#### 区域标题

```
"我的能力(6)"
字号: 14px, weight: 600, color: #1a1a2e
margin-bottom: 16px
```

#### 能力卡片（6 张，纵向排列，gap: 10px）

**尺寸**：宽度撑满，高度自适应（约 72-90px）

**布局**：

```
┌──────────────────────────────────┐
│ [图标]  能力名                     │
│         子能力1: 模型名   ●        │
│         子能力2: 模型名   ○        │
│         ─────────────────        │
│         Search it Plus            │
└──────────────────────────────────┘
```

**样式**：

- 背景：`rgba(255,255,255,0.7)` + `backdrop-filter: blur(12px)`
- 边框：`1px solid rgba(0,0,0,0.06)`
- 圆角：`12px`
- padding：`14px 16px`
- hover：`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`
- cursor：`pointer`

**图标**：每个能力用彩色渐变圆形图标（Figma 中可见）

- 聊天：橙色渐变 💬
- 画图：粉紫渐变 🎨
- 编程：蓝色渐变 💻
- 视频：绿色渐变 📹
- 语音：紫色渐变 🎙
- 记忆：青色渐变 🧠

**状态指示**：

- 已配置子能力：绿色圆点 `●` (#34d399)
- 未配置子能力：灰色圆点 `○` (#d1d5db)

**点击行为**：

- 点击能力卡 → 弹出**现有的模型选择器弹窗**（`openModelSelector()`）
- 多子能力卡片（图片/视频/语音/记忆）→ 弹窗内自动显示 Tab

### 3.2 右侧 — 模型配置（自适应宽度）

- padding：`16px`

- padding：`18px 20px`

## 六、实施步骤

### Step 1：CSS 全量重写

- 删除暗色 static styles
- 写入浅色毛玻璃主题
- 双栏布局（CSS Grid：`grid-template-columns: 320px 1fr`）
- 能力卡、Provider 卡、推荐卡样式
- 弹窗配色覆盖
- 排序浮动框样式

### Step 2：render() 主体重写

```typescript
render() {
  return html`
    <div class="mc-page">
      <!-- 左栏：能力 -->
      <div class="mc-left">
        <div class="mc-section-title">我的能力(${count})</div>
        ${this._renderCapabilityList()}    // 新：6 张纵向卡片
      </div>

      <!-- 右栏：Provider -->
      <div class="mc-right">
        <div class="mc-right-header">
          <div class="mc-section-title">模型配置</div>
          <button class="mc-add-btn" @click=${...}>+ 添加</button>
        </div>
        ${this._renderRecommendedProviders()}  // 新：顶部 3 张推荐
        ${this._renderConfiguredProviders()}    // 新：已配置大卡片
        ${this._renderUnconfiguredProviders()}  // 新：未配置网格
      </div>
    </div>

    <!-- 弹窗层（保留现有） -->
    ${this._s.modelSelectorOpen ? this._renderModelSelector() : nothing}
    ${this._s.providerConfigOpen ? this._renderProviderConfig() : nothing}
    ${this._s.providerManageOpen ? this._renderManageModal() : nothing}
    ${this._prioritySortOpen ? this._renderPrioritySortPopover() : nothing}
  `;
}
```

### Step 3：子渲染函数改写

- `_renderCapabilityList()` — 替代 `_renderCapabilities()`
- `_renderRecommendedProviders()` — 替代 `_renderOnboarding()`
- `_renderConfiguredProviders()` — 替代 `_renderMyProviders()`
- `_renderUnconfiguredProviders()` — 替代 `_renderAddProviders()`
- `_renderPrioritySortPopover()` — 新增排序浮动框
- 删除：`_renderOnboarding()`, `_renderEssentialBanner()`, `_renderQuickSwitch()` 系列, `_renderCapCard()` 中的 strength-warn

### Step 4：能力卡点击逻辑修改

- 删除快速切换面板逻辑（`_quickSwitchCap`, `_quickSwitchModels` 等状态）
- 能力卡 `@click` → 直接调用 `openModelSelector(h, capability)`
- 多子能力 → 弹窗内 Tab 切换

### Step 5：排序浮动框实现

- 新增状态：`_prioritySortOpen: boolean`
- 保留现有 `reorderProviders()` / `saveProviderPriority()` 逻辑
- 拖拽逻辑从主页面移到浮动框内
- 增加 ↑↓ 按钮作为辅助

### Step 6：弹窗配色变量替换

- 全局搜索暗色变量 → 替换为浅色值
- 或使用 CSS 变量做主题切换
